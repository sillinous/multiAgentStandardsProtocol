# APQC PCF Complete Hierarchy - All 5 Levels

## Overview

This document provides a comprehensive mapping of the APQC Process Classification Framework (PCFÂ®) across all 5 levels. The framework is used for atomic agent generation, where each Level 5 activity becomes a specialized agent.

**Framework Version**: APQC PCF 7.0.1
**Total Categories (Level 1)**: 13
**Total Process Groups (Level 2)**: ~50
**Total Processes (Level 3)**: ~150
**Total Activities (Level 4)**: ~500
**Total Tasks (Level 5)**: ~1,000+

## Hierarchical Structure

- **Level 1**: Operating Categories (13 categories)
- **Level 2**: Process Groups
- **Level 3**: Processes
- **Level 4**: Activities
- **Level 5**: Tasks (atomic - mapped to agents)

# Agent Generation Mapping

## Mapping Strategy

Each **Level 5 task** is mapped to an **atomic agent** with the following characteristics:

1. **Single Responsibility**: Each agent performs exactly one Level 5 task
2. **Domain Expertise**: Agent is specialized for its APQC category
3. **Composable**: Agents can be orchestrated into workflows
4. **Standardized**: All agents follow the same architectural pattern

## Agent Naming Convention

```
{Level5TaskName}{CategoryName}Agent

Examples:
- AnalyzeCompetitionStrategyAgent (1.1.1.1)
- ForecastDemandSupplyChainAgent (4.1.3.1)
- CalculateGrossPayFinancialAgent (9.5.2.1)
```

Use this hierarchy to:
1. Identify all Level 5 tasks
2. Generate atomic agent for each task
3. Group agents by category for testing
4. Build workflow templates combining agents

Level 5 tasks are the atomic building blocks. Combine them to create:
- **Intra-category workflows**: Using agents from same category
- **Cross-category workflows**: Using agents from multiple categories
- **End-to-end processes**: Complete business processes
